# Contributing to RealApi

Thanks for your interest! Contributions are welcome.

## How to Contribute
1. Fork the repo.
2. Create a branch (e.g., `feature/new-endpoint` or `bugfix/issue-1`).
3. Make changes, commit with clear messages.
4. Push to your fork.
5. Open a Pull Request to main with description.

## Guidelines
- For bugs/features, open an Issue first.
- Follow code style (use ESLint if setup, consistent indentation).
- Test changes (Postman for API, add to docs if new endpoints).
- Keep PRs focused (one feature/fix per PR).

Questions? Comment on Issues or PRs.
